ShelfScan will provide a collection view page at '/collection' that allows users to see all the 
games in their collection, along with their UPCs and any matched BGG data. This view is accessible from the main
menu and provides a convenient way for users to manage their collection and identify any missing or
unmatched games. Users can also use this view to quickly access game details and edit their
collection as needed.

The collection view will utilize the GameListContainer and ListGame components for display that 
matches the existing ScanList on the overview page.

It will also include features such as sorting and filtering options to help users find specific games in their collection.
The design will be consistent with the overall aesthetic of the ShelfScan app, ensuring a seamless user experience.

The list of games in the collection view will be generated based on the user's collection data, which is stored in the app's database.
It will utilize the 'react-virtuoso' virtualized list library
to efficiently render large collections and ensure smooth performance.

## Data Model & Source

A tab in the collection view will show all games which have been scanned which do not appear in 
the user's collection. This will be generated by comparing the list of scanned games (from the scan history) with the list of games in the user's collection (from the collection data).
Any games that appear in the scan history but not in the collection data will be displayed in this tab, allowing users to easily identify and add missing games to their collection.

The user will be responsible for refreshing their collection data from BGG to ensure that the collection view is up-to-date and accurately reflects their current collection. This can be done by using the existing "Refresh Collection" feature in the app, which will fetch the latest collection data from BGG and update the collection view accordingly.

For now, only the fields shown in the ListGame component will be included in the collection view,
but additional fields can be added in the future as needed. The collection view will be designed to be flexible and extensible, allowing for easy addition of new features and data fields as the app evolves.

## Filtering

Filtering on the name of the game will be the only filter available in the initial implementation, but additional filters can be added in the future as needed. The filtering will be implemented using a simple text input that allows users to search for games by name. As users type in the search input, the collection view will dynamically update to show only the games that match the search query, making it easier for users to find specific games in their collection.

## Sorting

Sorting based on the name of the game, the date it was added to the collection, the date it was 
last scanned, and the year it was published will be available in the initial implementation. Users will be able to sort the games in their collection by clicking on the corresponding column headers in the collection view. Clicking on a column header will toggle between ascending and descending order for that column, allowing users to easily organize their collection based on their preferences. Additional sorting options can be added in the future as needed, providing users with more flexibility in how they view and manage their collection.

## Editing

No editing actions will be supported in the initial implementation of the collection view, but this can be added in the future as needed. The collection view will primarily serve as a read-only interface for users to view and manage their collection, with editing functionality being added in a later iteration of the app.

## Actions

Actions available in the collection view will be to click on a game to open the game's page on 
BGG in a new tab, and to click on a "Refresh Collection" button to update the collection data from BGG. Clicking on a game in the collection view will open the game's page on BGG in a new browser tab, allowing users to quickly access detailed information about the game. The "Refresh Collection" button will allow users to fetch the latest collection data from BGG and update the collection view accordingly, ensuring that they have the most up-to-date information about their collection. Additional actions can be added in the future as needed, providing users with more ways to interact with their collection and manage their games effectively.

## Maximum Collection Size

The maximum collection size that the collection view will support in the initial implementation 
is 10,000 games.  Average collections are expected to be around 300-500 games.
